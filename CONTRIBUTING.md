# Contributing

Merci de l'intérêt pour contribuer à `ESP_Horloge_interne` ! Voici quelques règles simples pour contribuer efficacement.

1. Signalez d'abord les bugs ou demandes d'amélioration via les Issues.

2. Forkez le dépôt et créez une branche nommée de la forme `feature/xxx` ou `fix/yyy`.

3. Faites des commits petits et atomiques avec des messages clairs (ex: `fix: corrige débordement timer`).

4. Ouvrez une Pull Request depuis votre branche vers `main` et décrivez:
   - Le but du changement
   - Les étapes pour reproduire (si bug)
   - Les tests réalisés

5. Respectez la licence GPLv3 du projet. En soumettant une PR vous acceptez que votre contribution soit
   intégrée sous la même licence.

6. Style et qualité:
   - C++/Arduino : conservez la lisibilité et les commentaires en français si présents.
   - Évitez d'ajouter des fichiers binaires au dépôt.

7. Tests : si possible, fournissez des instructions de test dans la PR (ex: plateforme, commandes PlatformIO).

Merci — vos contributions sont les bienvenues !
